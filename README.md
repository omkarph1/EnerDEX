# EnerDEX

> **A full-stack, decentralised peer-to-peer renewable energy trading platform built on Ethereum.**

EnerDEX enables energy producers and consumers to trade tokenised energy units (ETK) directly on-chain, with automated fee collection, loyalty rewards, and real-time frontend synchronisation.

---

## Overview

The repository is split into two coordinated packages:

| Package | Description |
|---|---|
| [`dem-project/`](./dem-project/) | Hardhat workspace вЂ” Solidity smart contracts, tests, and deployment scripts |
| [`dem-frontend/`](./dem-frontend/) | React dApp вЂ” Vite + Ethers v6 user interface |

---

## Core Features

- **ERC-20 Energy Token (`ETK`)** вЂ” owner-controlled minting; 1 ETK represents 1 kWh of energy
- **Escrow Marketplace** вЂ” sellers lock ETK into the contract and receive ETH on successful purchase
- **Platform Fee** вЂ” 2% base fee on every trade, collected and withdrawable by the platform owner
- **Loyalty Reward System** вЂ” buyers and sellers each earn 10 points per trade
- **Tiered Fee Discounts** вЂ” applied automatically to buyers based on accumulated loyalty points:

  | Points | Discount |
  |---|---|
  | в‰Ґ 10 | 10% |
  | в‰Ґ 50 | 25% |
  | в‰Ґ 100 | 50% |

- **Automated Frontend Sync** вЂ” deployment script copies ABIs and writes contract addresses directly into the frontend environment
- **Event-Driven History** вЂ” `EnergyListed` and `EnergyTraded` events power the transaction history and seller income analytics screens

## Prerequisites

- **Node.js** v18 or later

- **npm** v9 or later

## End-to-End Local Setup

### Step 1 вЂ” Start the local Hardhat blockchain node

```bash
cd dem-project
npm install
npm run node
```

> Keep this terminal running. Hardhat will print 20 pre-funded test accounts and their private keys.

---

### Step 2 вЂ” Deploy contracts and sync the frontend (new terminal)

```bash
cd dem-project
npm run deploy
```

This single command:
1. Compiles and deploys `EToken` and `EnergyMarketplace`
2. Mints 500 ETK to the second Hardhat account (`signers[1]`) for use as a test seller
3. Copies compiled ABIs into `dem-frontend/src/contracts/`
4. Writes `dem-frontend/.env` with the deployed contract addresses

---

### Step 3 вЂ” Start the frontend (new terminal)

```bash
cd dem-frontend
npm install
npm run dev
```

Open the Vite dev server URL printed in the terminal, connect MetaMask to **Localhost 8545**, and import a Hardhat test account.

## Integration Notes

- `dem-frontend/.env` is generated automatically by the deploy script and is **gitignored**. It must be regenerated whenever contracts are redeployed.
- ABI files in `dem-frontend/src/contracts/` are also auto-generated; do not edit them manually.
- `EToken.mint(address, amount)` accepts **human units** (e.g. `500`); the contract scales by `10^18` decimals internally.
